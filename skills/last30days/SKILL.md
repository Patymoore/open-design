# Last 30 Days

Use this skill when the user cares about recency: launches, community reactions, GitHub momentum, product sentiment, new alternatives, fresh complaints, or "what changed this month?"

## Direct Run

```bash
node --experimental-strip-types scripts/last30days-lite.ts "topic" --limit 8
```

The included lite runner uses free-first sources:

- Web search through Brave when `BRAVE_API_KEY` is configured, otherwise DuckDuckGo HTML.
- Hacker News Algolia API for recent HN mentions.
- GitHub Search API for recently updated repositories.
- Site-scoped searches for Reddit, YouTube, TikTok, X, and other public pages.

For the full upstream engine, install `https://github.com/mvanhorn/last30days-skill` into the active agent skills directory. This built-in skill keeps the workflow discoverable in Open Design and gives a no-key fallback that works immediately.

## Agent Workflow

1. Define the 30-day window explicitly in the answer.
2. Run the lite script, then follow up with targeted `search` queries when coverage is thin.
3. Separate community chatter, source-code momentum, news/blog coverage, and official announcements.
4. Cite URLs inline. Do not overstate search snippets as measured sentiment.
5. End with patterns, counter-signals, and what the user should do next.
